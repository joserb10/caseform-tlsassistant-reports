Cipher Suites TLS 1.3 compliance

Compliance report for CipherSuitesTLS1.3 against BSI guidelines

Mitigation
  • Textual

    Add the following cipher suites to the server configuration:
    - TLS_AES_128_CCM_SHA256 should be enabled according to BSI
    If TLS 1.3 is not enabled the tool can not check the compliance of the cipher suites mentioned above.

  • Apache

    - Quick solution: replace the instruction of the directive SSLOpenSSLConfCmd Ciphersuites with the following string:
    TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_AES_128_CCM_SHA256;
    - Advanced solution: check the directive SSLOpenSSLConfCmd Ciphersuites and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element you can add either - (remove once) or ! (kill) in front of it.

  • Nginx

    - Quick solution: replace the instruction of the directive ssl_conf_command Ciphersuites with the following string:
    TLS_AES_128_GCM_SHA256:TLS_AES_256_GCM_SHA384:TLS_AES_128_CCM_SHA256;
    - Advanced solution: check the directive ssl_conf_command Ciphersuites and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element you can add either - (remove once) or ! (kill) in front of it.

    - TLS_ECDHE_PSK_WITH_AES_256_CBC_SHA384 should be enabled according to BSI
    - TLS_ECDHE_PSK_WITH_AES_128_GCM_SHA256 should be enabled according to BSI
    - TLS_ECDHE_PSK_WITH_AES_256_GCM_SHA384 should be enabled according to BSI
    - TLS_ECDHE_PSK_WITH_AES_128_CCM_SHA256 should be enabled according to BSI
    - TLS_DHE_PSK_WITH_AES_128_CBC_SHA256 should be enabled according to BSI
    - TLS_DHE_PSK_WITH_AES_256_CBC_SHA384 should be enabled according to BSI
    - TLS_DHE_PSK_WITH_AES_128_GCM_SHA256 should be enabled according to BSI

    - TLS_DHE_PSK_WITH_AES_128_CCM should be enabled according to BSI
    - TLS_DHE_PSK_WITH_AES_256_CCM should be enabled according to BSI

Groups compliance

Compliance report for Groups against BSI guidelines

Mitigation
  • Textual

    Add the following curves to the server configuration:
    - ffdhe3072 should be enabled according to BSI
    - ffdhe4096 should be enabled according to BSI
    - brainpoolP256r1 should be enabled according to BSI
    - brainpoolP384r1 should be enabled according to BSI
    - brainpoolP512r1 should be enabled according to BSI
    - brainpoolP256r1tls13 should be enabled according to BSI
    - brainpoolP384r1tls13 should be enabled according to BSI
    - brainpoolP512r1tls13 should be enabled according to BSI

  • Apache

    - Quick solution: replace the instruction of the directive SSLOpenSSLConfCmd groups with the following string:
    secp256r1:secp384r1:secp521r1:ffdhe3072:ffdhe4096:brainpoolP256r1:brainpoolP384r1:brainpoolP512r1:brainpoolP256r1tls13:brainpoolP384r1tls13:brainpoolP512r1tls13;
    - Advanced solution: check the directive SSLOpenSSLConfCmd groups and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element it is necessary to remove it from the string.
    NOTE: in order to disable ffdhe* groups remove the dhparams from the end of the first certificate file

  • Nginx

    - Quick solution: replace the instruction of the directive ssl_ecdh_curve with the following string:
    secp256r1:secp384r1:secp521r1:ffdhe3072:ffdhe4096:brainpoolP256r1:brainpoolP384r1:brainpoolP512r1:brainpoolP256r1tls13:brainpoolP384r1tls13:brainpoolP512r1tls13;
    - Advanced solution: check the directive ssl_ecdh_curve and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element it is necessary to remove it from the string.
    NOTE: in order to disable ffdhe* groups remove the ssl_dhparam directive.
    NOTE: this directive is only supported by Nginx 1.19.4 or higher.

Signature Algorithms compliance

Compliance report for Signature against BSI guidelines

Mitigation
  • Textual

    Enable the following signature algorithms:
    - rsa_pss_rsae_sha256 should be enabled according to BSI
    - rsa_pss_rsae_sha384 should be enabled according to BSI
    - rsa_pss_rsae_sha512 should be enabled according to BSI
    - rsa_pss_pss_sha256 should be enabled according to BSI
    - rsa_pss_pss_sha384 should be enabled according to BSI
    - rsa_pss_pss_sha512 should be enabled according to BSI
    - ecdsa_secp521r1_sha512 should be enabled according to BSI
    - ecdsa_brainpoolP256r1tls13_sha256 should be enabled according to BSI
    - ecdsa_brainpoolP384r1tls13_sha384 should be enabled according to BSI
    - ecdsa_brainpoolP512r1tls13_sha512 should be enabled according to BSI
    NOTE: your OpenSSL version may not support all the signature algorithms listed in this section. The mitigations provided in this report only show the supported ones.

  • Apache

    - Quick solution: replace the instruction of the directive SSLOpenSSLConfCmd ClientSignatureAlgorithms with the following string:
    rsa_pss_rsae_sha256:rsa_pss_rsae_sha384:rsa_pss_rsae_sha512:rsa_pss_pss_sha256:rsa_pss_pss_sha384:rsa_pss_pss_sha512:ecdsa_secp256r1_sha256:ecdsa_secp384r1_sha384:ecdsa_secp521r1_sha512;
    - Advanced solution: check the directive SSLOpenSSLConfCmd ClientSignatureAlgorithms and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element you have to remove it from the string.

  • Nginx

    - Quick solution: replace the instruction of the directive ssl_conf_command SignatureAlgorithms with the following string:
    rsa_pss_rsae_sha256:rsa_pss_rsae_sha384:rsa_pss_rsae_sha512:rsa_pss_pss_sha256:rsa_pss_pss_sha384:rsa_pss_pss_sha512:ecdsa_secp256r1_sha256:ecdsa_secp384r1_sha384:ecdsa_secp521r1_sha512;
    - Advanced solution: check the directive ssl_conf_command SignatureAlgorithms and for each element in the Textual section above, follow the instructions provided. The separator for this directive is : and to disable an element you have to remove it from the string.


    NOTE: this directive is only supported by Nginx 1.19.4 or higher.
